Configure File

All of the configurations for the default log file can be configured with this one VI instead of using VIs for each individual configuration. The individual configuration VIs are just wrappers around this VI.

Error In describes error conditions that occur before this node runs. This input provides standard error in functionality.

Configuration is all of the configurations for the default log file.

    Root Folder is the folder where log files will be created and saved. The path cannot be empty or Not a Path.

    Enable Calendar Tree determines if the log files are organized into a calendar-based folder heirarchy based on the creation date of the log file.

    Maximum File Size is the maximum file size in bytes for a log file.

    Maximum File Count is the maximum number of log files to keep on-disk before deleting the oldest log file. A zero (0) indicates no log files will ever be deleted.

    File Extension The extension to use for the log file name. The default is "csv". A leading period is optional.

Error Out contains error information. This output provides standard error out functionality.
